# 자동 발행 작업 지시서 (스포츠킹덤24)

이 문서는 예약 루틴이 **이틀에 한 번** 따라 실행하는 절차다. 목표는 단순 발행이 아니라, **해당 키워드로 검색했을 때 상위 글들보다 더 충실하고, 구글 SEO에 맞게 정리된 글 1편**을 내보내는 것이다. 각 글에 외부링크 1개·내부링크 1개·이미지 1개가 들어가고, **발행 전 반드시 웹 팩트체크**한다.

## 절차 (4단계)

작업 폴더는 저장소 루트(`gameworld7`). PATH에 node가 없으면 먼저 잡는다.

### 1단계 — 재료 받기
```
node automation/prepare.mjs
```
출력 JSON(= `automation/assignment.json`): `topic`, `pubDate`, `heroImage`, `internalLink{url,title}`, `externalLink{url,label}`.
"발행할 주제가 없습니다"로 멈추면 글을 쓰지 말고 **즉시 종료**(빈 글 금지).

### 2단계 — 경쟁글 깊이 분석 (글 쓰기 전 필수)
`topic`(키워드)을 **WebSearch로 검색하고, 상위 10개 중 최소 5개 이상을 WebFetch로 직접 열어** 분석한다. 다음을 파악한다.
- **검색의도** — 이 키워드를 친 사람이 진짜 원하는 게 정보인지, 추천 목록인지, 방법(How-to)인지, 다운로드/사이트인지.
- **상위 글들의 구성** — 어떤 소제목(H2)들을 다루는가, 평균 분량은 어느 정도인가.
- **공통 커버 항목 체크리스트(필수)** — 상위 글 과반에 공통으로 나오는 항목을 목록으로 뽑는다. 이 목록은 **하나도 빠짐없이 우리 글에 포함**한다. 상위권이 공통으로 다루는데 우리만 빠지면 검색의도 충족에서 밀린다.
- **빠진 것(콘텐츠 갭)** — 상위 글들이 다루지 않았거나 부실하게 다룬 부분. 우리가 더 잘 쓸 지점(차별화).

이를 바탕으로 **상위 글이 다루는 핵심을 모두 덮으면서 + 빠진 부분을 채우는** 아웃라인을 먼저 짠다. 사실 정보(팀·선수 이름·기록·일정·규칙 등)는 이 과정에서 같이 확인한다.

### 3단계 — 글 작성
`src/content/blog/<영문-slug>.md` 작성. 아래 **품질·SEO 규칙**을 지킨다.

### 4단계 — 팩트체크 → 발행
사실 주장을 다시 검증(아래 팩트체크 규칙) 후:
```
node automation/finalize.mjs <slug>
```
링크·이미지·최소 분량·소제목 수 검증 + 빌드 통과 시 자동 커밋·푸시. 실패하면 메시지대로 보강해 재실행.

## 품질·형식 규칙 (반드시)

- 본문 **최소 2,500자**(한글, 권장 2,500~3,500), 그리고 **상위 경쟁글 평균보다 충실하게**. finalize가 미달이면 막는다.
- `##` 소제목 **7~10개**, 각 섹션 3문단 이상으로 깊이 있게. 비교 표·단계별 목록·체크리스트·실전 예시/시나리오·**FAQ(4개 이상)**를 적극 활용해 길이와 가치를 함께 채운다.
- **확장 패스(필수):** 초안을 쓴 뒤 각 섹션을 다시 읽고, 2문단 미만이거나 일반론에 그친 부분은 구체적 예시·근거·수치·비교로 보강한다. 목표 분량에 못 미치면 빈말로 늘리지 말고, 경쟁글이 다뤘는데 빠진 소주제를 새 섹션으로 추가한다.
- 빈말("재밌습니다", "좋아요") 금지. 구체적 예시·이유·실제 팀/선수/종목 사례로 채운다. 독자가 이 글 하나로 이해·행동할 수 있을 만큼 친절·구체적으로.
- 톤: 존댓말 + 사람 냄새, 직접 해본 사람이 풀어주듯. 정형 라벨 나열 금지.
- 한국어 문장을 콜론(`:`)으로 끝내지 않는다. `.`, `?`, `!`로 끝낸다.

## 검색엔진 최적화(SEO) 규칙 (반드시)

- **제목(title)**: `topic` 키워드를 **앞쪽에** 자연스럽게 넣는다. 클릭하고 싶게, 60자 이내 권장.
- **설명(description)**: 키워드를 포함한 120~155자 요약. 검색결과에 그대로 노출되니 클릭을 유도하게 쓴다.
- **키워드 배치**: 제목(H1)·**첫 문단(앞 100자 안)**·최소 1개 `##` 소제목·본문에 자연스럽게 등장시킨다. 단, 부자연스러운 반복(키워드 스터핑) 금지 — 동의어·연관어도 함께 쓴다.
- **구조**: `##`/`###` 계층으로 스캔 가능하게. 핵심 답은 글 앞부분에서 먼저 준다(검색 의도 빠른 충족).
- **내부링크 1개**: `[자연스러운 앵커](internalLink.url)`. **외부링크 1개**: `[자연스러운 앵커](externalLink.url)`.
- **이미지**: frontmatter `heroImage`만 지정(레이아웃이 alt·OG·배너를 자동 처리). 본문에 따로 넣지 않는다.
- **slug**: 키워드를 영문/로마자로 옮긴 짧고 의미 있는 kebab-case.
- **본문 끝 관련 키워드(필수)**: 글 맨 마지막에 `---` 구분선 뒤 한 줄로 관련 키워드 묶음을 넣는다. 형식 `**관련 키워드** — #키워드1 #키워드2 …`(타겟·동의어·세부 변형 8~12개, 띄어쓰기 없는 해시태그). 타겟과 실제로 관련된 것만 넣고 무관한 키워드 스터핑은 금지.
- **표·콜아웃 박스로 가독성(권장)**: 비교·정리는 마크다운 표로 쓴다(레이아웃이 자동으로 보기 좋게 스타일). 핵심 요약·주의·팁은 콜아웃 박스를 쓴다. 박스는 raw HTML로 쓰되 **안쪽도 마크다운이 아닌 HTML 태그**로 작성한다(div 안에서는 마크다운이 렌더되지 않음). 글 맨 위에 `cg-summary` 3줄 요약 박스 1개를 권장.
  - 형식: `<div class="cg-box cg-summary"><span class="cg-title">📌 3줄 요약</span><p>…</p><p>…</p></div>`
  - 종류: `cg-summary`(요약·오렌지) · `cg-tip`(팁·초록) · `cg-warn`(주의·노랑) · `cg-danger`(위험·빨강). 박스 스타일은 `src/layouts/BlogPost.astro`에 정의돼 있다.
- frontmatter 형식:
  ```markdown
  ---
  title: '키워드를 앞에 넣은 한국어 제목'
  description: '키워드 포함 120~155자 요약'
  pubDate: '<assignment의 pubDate 그대로>'
  heroImage: '<assignment의 heroImage 그대로>'
  ---
  ```

## 팩트체크 규칙 (발행 전 반드시)
- **(필수) 수치·규정·기록·날짜·가격은 1차 출처(공식 사이트) 또는 상위 글 2곳 이상을 직접 열어(WebFetch) 교차확인한다.** 검색 결과 스니펫 요약만 보고 쓰지 않는다.
- **출처마다 값이 다르면 단정 금지** — 공통으로 일치하는 값만 쓰거나 "출처마다 다름, 공식 발표 기준 확인"으로 표기하고 상충 수치는 뺀다.
- 글의 **모든 구체적 사실**(팀·선수 이름·기록·순위·일정·규칙·수치)을 웹으로 확인한다.
- 확인 안 되면 삭제하거나 단정하지 않는 표현으로 바꾼다("보통", "대체로").
- **잘 바뀌는 정보는 단정 금지** — 정확한 가격·할인율·날짜·세부 사양은 "정가 기준", "세일 때 더 저렴" 식으로. "상시 ○○원" 금지.
- 출처 불분명한 수치·통계는 쓰지 않는다. **의심스러우면 뺀다**(틀린 정보가 빈약한 정보보다 나쁘다).

## 🥇 상위노출 전략 — SERP 해부 · 10x 보강 · 승산 키워드

신생 도메인은 권위가 약해, 경쟁 센 머리 키워드는 콘텐츠만으로 1위가 어렵다. 현실 전략은 **① 이길 수 있는 SERP를 고르고 → ② 그 키워드에서 상위 10개보다 압도적으로 충실한 글 → ③ 토픽 클러스터로 내부 권위를 모으는 것**이다.

### A. SERP 해부 (2단계 분석을 표로 구조화)
상위 10개를 열어 아래를 표로 적고, 마지막 두 칸에서 '우리 노림수'를 뽑는다.

| 항목 | 기록 내용 |
| --- | --- |
| 검색의도 | 정보 / 추천목록 / How-to / 사이트·다운로드 중 무엇인가 |
| 공통 필수항목 | 과반이 공통으로 다루는 소제목 — **하나도 빠짐없이 포함** |
| 각 글의 강점 | 상위권이 잘한 점(우리도 최소한 충족) |
| 각 글의 약점·누락 | 부실하거나 빠뜨린 것 = **우리 차별화(1위 노림수)** |
| 평균 분량 | 상위 평균보다 충실하게(단, 빈말로 늘리지 말 것) |
| SERP 특징 | 추천 스니펫 · PAA(함께 묻는 질문) · 이미지팩 유무 |

### B. 10x 보강 요소 (경쟁글이 잘 안 넣는 것 — 적극 추가)
- **직접 경험·구체 시나리오** — "이럴 땐 이렇게"(구글 E-E-A-T의 Experience).
- **의사결정 도구** — 비교표·체크리스트·플로차트·간단 계산.
- **PAA 정조준 FAQ** — '함께 묻는 질문'을 그대로 FAQ로(템플릿이 FAQPage 스키마를 자동 생성한다).
- **1차 출처 인용 + 최신성** — 공식 수치 + "○○년 기준" 표기.
- **추천 스니펫 겨냥** — 각 H2의 핵심 답을 첫 2~3문장에 압축(구글이 그 문단을 뽑아간다).

### C. 승산 키워드 고르기 (점수가 높을수록 1위 가능)
주제를 topics.md에 넣기 전, 검색해서 상위 10개 SERP를 보고 아래로 가늠한다.

| 신호 | +승산 | −승산 |
| --- | --- | --- |
| 상위권 구성 | 개인블로그·카페·포럼·얇은 글 많음 | 공식기관·대형매체·위키 많음 |
| 콘텐츠 깊이 | 상위도 부실(보강 여지 큼) | 이미 매우 충실 |
| 키워드 길이 | 구체적 롱테일(3어절+) | 짧은 머리 키워드 |
| 검색의도 | 정보·How-to(글로 충족) | 거래·브랜드·사이트 이동 |

→ 왼쪽(+)이 많은 키워드부터 공략한다. 머리 키워드는 권위가 쌓인 뒤로 미룬다. **즉시 1위는 환상이고, 승산 롱테일을 쌓아 트래픽·권위를 만드는 게 정석.**

### D. 클러스터 내부링크
같은 주제군 글끼리 서로 링크해 권위를 모은다(내부링크를 '아무 글'이 아니라 **같은 주제의 글**로 건다). prepare가 배정한 내부링크가 주제와 안 맞으면 더 관련 있는 기존 글로 바꿔도 된다.

## 주제가 떨어졌을 때
prepare가 "발행할 주제가 없습니다"로 멈추면 발행하지 말고 종료(빈 글 금지). 주제는 사람이 `automation/topics.md`에 채운다.
